## 2025-02-28 - Converting Interactive List `div` Elements to `<button>`
**Learning:** When dealing with legacy components that use standard `<div>` elements as interactive, clickable list items (e.g., notification alerts), merely adding `onClick` lacks semantic meaning and critical keyboard/screen reader accessibility. However, converting them to `<button>` can break expected layouts if styles are inherited. The solution is applying specific inline CSS resets (`background: 'none'`, `border: 'none'`, `width: '100%'`, `textAlign: 'left'`) to maintain the existing layout and visual fidelity. Additionally, mapping `onFocus` and `onBlur` handlers identically to `onMouseEnter` and `onMouseLeave` ensures proper visible indication during keyboard navigation, matching the mouse hover experience. Adding an `aria-label` based on the item's content vastly improves screen reader usability.
**Action:** When migrating interactive list items from `<div>` to `<button>`, apply the necessary inline style resets and map hover states to focus states to maintain visual parity while strictly enforcing semantic HTML and a11y standards. Ensure proper `aria-label` injection if the internal content is complex.
